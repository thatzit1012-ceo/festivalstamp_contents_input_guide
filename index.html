스탬프 콘텐츠
입력 가이드

엑셀 입력폼에 행사 정보와 스탬프(장소·콘텐츠) 정보를 적고, 이미지와 함께 묶어 한 번에 등록하는 방법을 처음부터 끝까지 안내합니다.

· 진행 순서

전체 과정은 네 단계입니다. 입력폼을 작성하고, 이미지를 준비해, 하나의 ZIP으로 묶어 업로드하면 됩니다.

1입력폼 작성엑셀에 정보 입력
→
2이미지 준비스탬프별 사진 정리
→
3ZIP 묶기엑셀 + images 폴더
→
4업로드관리자 페이지 등록
이미지가 없어도 등록은 가능합니다. 사진이 없는 스탬프에는 ‘이미지 준비중’ 기본 이미지가 자동으로 들어갑니다.

1 입력폼(엑셀) 작성

양식은 관리자 페이지에서 내려받거나, 위의 예시 입력폼을 받아 그대로 고쳐 쓰면 편리합니다. 엑셀은 시트 2장으로 구성됩니다 — ① 행사 정보 시트에 축제 소개와 선물 정보를, ② 스탬프 콘텐츠 시트에 스탬프(장소)를 한 줄씩 입력합니다.

시트 ① — 행사 정보

축제 자체에 대한 정보입니다. 시트의 회색 칸을 지우고 실제 내용을 채워 주세요. (예)로 시작하는 글은 예시입니다.

항목설명
행사명필수 축제(행사)의 공식 이름입니다. 앱과 관리자 화면에 그대로 표시됩니다.예) 전국 시장 맛집 스탬프투어
행사기간필수 행사 시작일과 종료일입니다.예) 2026-10-01 ~ 2026-10-31
장소선택 행사가 열리는 장소·지역입니다.예) 남대문·동대문·경동시장 일원
주최선택 행사를 주최·주관하는 기관명입니다.예) 소상공인시장진흥공단
홈페이지 · 행사장 안내도선택 행사 공식 홈페이지 URL과 행사장 안내도입니다. 안내도는 URL을 적거나, 이미지 파일을 ZIP의 images/map.png로 넣어 주세요.
스탬프 이벤트 내용선택 이벤트를 소개하는 문구입니다. 앱 정보 화면에 노출됩니다.예) 시장 맛집을 방문해 QR 스탬프를 모으면 선물을 드립니다
선물 정보필수 선물을 받기 위한 스탬프 개수와 선물 내용을 적습니다. 선물이 2가지면 두번째 선물 칸(선택)도 채웁니다.예) 스탬프 5개 → 시장 상품권 5,000원 / 스탬프 10개 → 시장 상품권 10,000원
선물 지급 장소 · 선물 공지선택 선물을 받는 장소와 지급 방식 안내입니다.예) 각 시장 고객지원센터 / 매일 선착순 100명 지급

시트 ② — 스탬프 콘텐츠

1행에 항목 이름이 들어 있으니 2행부터 한 줄에 스탬프 하나씩 입력합니다. 항목은 모두 7개입니다.

항목설명
순번필수 1부터 차례대로 매기는 번호입니다. 이미지 파일명과 반드시 일치시켜야 사진이 올바르게 연결됩니다.예) 1, 2, 3, 4 …
스탬프명필수 앱 화면에 표시되는 장소·콘텐츠의 이름입니다. 영문(다국어) 화면에도 함께 노출하려면 한글명(영문명) 형식으로 적습니다.예) 효자왕만두(hyojawangmandoo)
설명선택 스탬프 상세 페이지에 보이는 소개 글입니다. 메뉴·위치·전화·영업시간처럼 항목을 나눠 적으면 보기 좋게 노출됩니다.예) 주요메뉴 : 손만두, 찐빵 위치 : 서울 중구 남창동34-126 전화번호 : 02-3789-7940 영업시간 : AM8:00~PM21:00
주소선택 장소의 도로명 또는 지번 주소입니다.예) 서울 중구 남창동 34-126
위도 · 경도선택 지도에 위치를 표시하기 위한 좌표입니다. 소수점 형태로 입력하며, 4자리 이상이면 충분히 정확합니다. 모르면 비워두어도 됩니다.예) 위도 37.5588 / 경도 126.9765
코스(course)조건부 스탬프를 여러 묶음으로 나누는 축제에서만 입력합니다. 자세한 사용법은 아래 코스 기능을 참고하세요. 나누지 않으면 비워둡니다.예) 남대문시장(namdaemunmarket) / 1코스

실제 입력 예시

예시 입력폼에 들어 있는 데이터의 첫 줄입니다. 이런 형태로 한 줄씩 채워 넣으면 됩니다.

순번스탬프명(필수)설명주소위도경도코스(course)
1 효자왕만두(hyojawangmandoo) 주요메뉴 : 손만두, 찐빵, 도너츠 / 전화 : 02-3789-7940 / 영업 : AM8~PM9 서울 중구 남창동34-126 37.5588 126.9765 남대문시장(namdaemunmarket)
작성할 때 유의할 점
  • 순번은 1부터 빠짐없이 이어지는 숫자로 — 이미지 파일명(1.png, 2.png …)과 같게 맞춥니다.
  • 스탬프명이 비어 있는 줄은 자동으로 건너뜁니다. (빈 줄로 구분해도 무방)
  • 입력한 줄 순서가 곧 앱에서 보이는 순서가 됩니다. 보여주고 싶은 순서대로 정렬하세요.
  • 엑셀 양식의 1행(항목 이름)은 지우거나 바꾸지 마세요.

2 코스(course) 기능

하나의 축제 안에서 스탬프를 여러 묶음(코스)으로 나눠 보여주는 기능입니다. 여러 구역·테마·시장을 한 축제에서 운영할 때 유용합니다.

엑셀의 코스(course) 칸에 같은 이름을 적은 스탬프끼리 한 묶음이 되고, 서로 다른 코스의 스탬프는 섞이지 않습니다. 예를 들어 여러 전통시장을 도는 축제라면 시장별로 코스를 나눠 한눈에 보여줄 수 있습니다.

묶음 예시 — ‘시장별 맛집’ 축제

📍 남대문시장
효자왕만두남대문숯불갈비대도식당진주집
📍 양동시장
양동통닭수민이네홍어복춘식당양동골목분식
📍 서문시장
서문칼국수진서방납작만두할매수제비명물씨앗호떡
📍 동대문시장
닭한마리엄마손칼국수동대문왕족발종로떡볶이

↑ 같은 코스명을 적은 스탬프끼리 묶여서 표시됩니다. (예시 입력폼에는 6개 시장 코스가 들어 있습니다)

코스 작성 규칙
  • 코스를 사용할 때 — 모든 줄에 코스명을 빠짐없이 적습니다. 빈 칸이 섞이면 묶음이 어긋날 수 있습니다.
  • 코스를 사용하지 않을 때 — 코스 칸을 전부 비워둡니다. 그러면 일반 스탬프투어처럼 하나의 목록으로 동작합니다.
  • 같은 코스는 코스명을 완전히 똑같이 적어야 묶입니다. (띄어쓰기·괄호·영문 표기까지 동일하게)
  • 코스명도 다국어 노출을 위해 한글명(영문명) 형식을 권장합니다.
⚠️ 한 글자만 달라도 다른 코스로 인식됩니다. 예를 들어 1코스 와 1 코스(가운데 공백)는 서로 다른 묶음이 됩니다. 한 셀을 복사해 같은 코스의 다른 줄에 붙여넣어 통일하는 것을 권장합니다.

3 이미지 준비

이 부분은 헷갈리기 쉬우니 자세히 설명합니다. 핵심부터 말하면 — 입력폼과 함께 올리는 이미지는 ‘스탬프 상세 페이지’에 들어가는 대표 사진 한 장입니다.

이 사진이 들어가는 곳

앱 안에는 위치별로 서로 다른 이미지가 쓰입니다. 아래 세 화면 중 세 번째(스탬프 상세 페이지)에 들어가는 사진만 이 입력폼으로 등록합니다. 홈 화면 목록과 축제 메인 배너 이미지는 입력폼과 별도로 설정합니다.

① 홈 화면 목록권장 359 × 130
별도 설정
② 축제 상세 메인권장 375 × 270
별도 설정
③ 스탬프 상세 페이지권장 375 × 281
★ 이 입력폼으로 올리는 사진

권장 크기

스탬프 상세 페이지의 사진은 375 × 281 비율(가로가 살짝 더 긴 직사각형)입니다. 화면에서 선명하게 보이도록 2~3배 크게 만드는 것을 권장합니다. 즉 실제 제작 크기는 약 750 × 560px ~ 1,125 × 840px 정도가 적당합니다.

정사각형(예: 800 × 800)으로 만들면 위아래가 잘려 보일 수 있습니다. 위의 가로형 비율(약 4:3)에 맞춰 주세요.

파일명 규칙

이미지 파일 이름을 순번.확장자 형식으로 지으면, 엑셀의 순번과 자동으로 연결됩니다.

순번 1→1.png1번 스탬프 상세 사진으로 등록
순번 2→2.jpg2번 스탬프 상세 사진으로 등록
순번 3→(파일 없음)‘이미지 준비중’ 기본 이미지 자동 적용
순번 4→4.jpeg4번 스탬프 상세 사진으로 등록

지원 형식 · 그 밖의 안내

  • 지원 형식: png · jpg · jpeg · gif · webp (확장자만 순번에 맞추면 형식은 무엇이든 괜찮습니다)
  • 코스를 나눠도 순번은 코스와 상관없이 1부터 전체 통합으로 매깁니다. (코스별로 다시 1번부터 시작하지 않습니다)
  • 사진이 없는 스탬프는 파일을 만들지 않으면 됩니다. 자동으로 ‘이미지 준비중’ 기본 이미지가 들어갑니다.
  • 사진이 너무 무거우면 업로드가 느려집니다. 가로 1,000px 내외로 줄이면 화질과 용량의 균형이 좋습니다.

4 ZIP으로 묶어 업로드

엑셀 입력폼과 이미지 폴더를 하나의 ZIP 파일로 묶어 관리자 페이지에서 업로드합니다.

ZIP 안의 구조

📦 upload.zip
├── 입력폼.xlsx   ← 엑셀 입력폼 (파일명 자유, ZIP 맨 위에)
└── 📁 images/   ← 폴더명은 반드시 영문 소문자 images
    ├── 1.png
    ├── 2.jpg
    ├── 3.png
    └── 4.jpeg

ZIP 만드는 방법

Windows — 엑셀 파일과 images 폴더를 함께 선택(Ctrl+클릭)한 뒤, 마우스 우클릭 → ‘압축(ZIP 폴더로)’ 또는 ‘보내기 → 압축(ZIP) 폴더’를 누릅니다.

Mac — 두 항목을 함께 선택한 뒤 우클릭 → ‘[2개 항목] 압축’을 누르면 아카이브.zip이 만들어집니다.

만들어진 .zip 파일을 관리자 페이지의 일괄 등록 화면에서 업로드하면 됩니다.

꼭 지켜 주세요
  • images 폴더 이름은 영문 소문자 그대로 써야 합니다. (Images, IMAGES, 이미지 ✗)
  • 엑셀 파일은 하위 폴더가 아니라 ZIP 맨 위(최상위)에 있어야 인식됩니다.
  • 폴더째로 한 번 더 압축되지 않도록, 엑셀과 images 폴더를 함께 선택해서 압축하세요.
⬇
예시 입력폼(엑셀) 내려받기 ‘시장별 맛집’ 축제 예시 데이터가 담긴 입력폼입니다(행사 정보 + 스탬프 콘텐츠 시트). 받아서 내용만 바꿔 쓰면 가장 빠릅니다.

? 자주 묻는 질문 (FAQ)

입력·등록 과정에서 자주 나오는 질문을 모았습니다. 항목을 누르면 답이 펼쳐집니다.

‘행사 정보’ 시트는 어디에 쓰이나요?
행사명·행사기간·주최·선물 정보 등은 운영팀이 축제를 등록할 때 그대로 반영합니다. 앱의 정보 화면과 선물 안내에 노출되는 내용이므로 빠짐없이 적어 주세요. 자동으로 등록되는 것은 [스탬프 콘텐츠] 시트의 스탬프들입니다.
코스 기능을 꼭 써야 하나요?
아니요. 스탬프를 묶어서 보여줄 필요가 없으면 코스 칸을 전부 비워두면 됩니다. 그러면 모든 스탬프가 하나의 목록으로 동작합니다. 시장·구역·테마별로 나눠 보여주고 싶을 때만 사용하세요.
일부 줄만 코스명을 적어도 되나요?
안 됩니다. 코스를 사용한다면 모든 줄에 코스명을 적어야 합니다. 빈 칸이 섞여 있으면 묶음이 제대로 처리되지 않을 수 있습니다. 코스를 쓰지 않을 거라면 칸 전체를 비워두세요.
같은 코스인데 다른 묶음으로 나뉘었어요.
코스명이 한 글자라도 다르면 다른 코스로 인식됩니다. 띄어쓰기·괄호·영문 표기까지 완전히 똑같이 적어야 한 묶음이 됩니다. 한 셀을 복사해 같은 코스의 다른 줄에 붙여넣어 통일하는 것을 권장합니다.
입력폼으로 올리는 이미지는 어느 화면에 나오나요?
스탬프를 눌렀을 때 나오는 ‘스탬프 상세 페이지’의 대표 사진으로 들어갑니다. 홈 화면 목록 이미지나 축제 메인 배너는 이 입력폼이 아니라 별도로 설정합니다. 위 ‘이미지 준비’의 화면 안내를 참고하세요.
이미지를 정사각형으로 만들어야 하나요?
아니요. 스탬프 상세 페이지는 가로가 조금 더 긴 375 × 281 비율입니다. 정사각형으로 만들면 위아래가 잘려 보일 수 있으니 이 비율에 맞추고, 선명하게 보이도록 2~3배 크게(약 750×560 ~ 1,125×840px) 제작하세요.
이미지 형식이 JPG인데 PNG로 바꿔야 하나요?
아니요. png · jpg · jpeg · gif · webp 모두 지원합니다. 파일 이름만 순번.확장자 형식(예: 2.jpg)으로 맞춰 주면 됩니다.
이미지 없이 입력폼만 올려도 되나요?
네, 가능합니다. images 폴더 없이 엑셀만 ZIP에 담거나, 사진이 없는 스탬프는 파일을 생략하면 됩니다. 사진이 없는 스탬프에는 ‘이미지 준비중’ 기본 이미지가 자동으로 들어갑니다.
코스를 나누면 이미지 순번도 코스별로 1번부터 시작하나요?
아니요. 코스와 상관없이 순번은 전체 통합으로 1번부터 이어집니다. 이미지 파일명도 그 통합 순번을 그대로 따릅니다.
위도·경도를 모르면 어떻게 하나요?
비워두어도 등록됩니다. 다만 지도에 위치가 표시되지 않습니다. 좌표는 포털 지도에서 장소를 검색해 ‘좌표 보기/공유’ 기능으로 확인할 수 있습니다.
이미 등록된 축제에 스탬프를 추가로 올려도 되나요?
추가는 가능합니다. 단, 추가한 항목은 기존 항목들 뒤에 붙습니다. 기존 항목 사이에 끼워 넣을 수는 없으니, 순서가 중요하다면 처음 등록 시 순서를 정해 두는 것이 좋습니다.
업로드 중 일부 항목만 실패하면 어떻게 되나요?
성공한 항목은 정상 등록되고, 실패한 항목만 결과 화면에 오류 내용이 표시됩니다. 그 항목만 수정해 별도 ZIP으로 다시 올리면 됩니다.
ZIP 파일 크기 제한이 있나요?
100MB 이하를 권장합니다. 이미지가 많으면 가로 1,000px 내외로 줄여 용량을 낮춰 주세요. 사진을 적절히 압축하면 수십 개도 무리 없이 한 번에 올릴 수 있습니다.
